# mt5-market-schedule

Machine-generated schedule of **early market closes** (US index + CME metals/energy)
and **ICMarkets server maintenance** windows, for consumption by an MT5 Expert Advisor
on a VPS.

The EA reads `early_closes.json` via `WebRequest` and flattens its own positions a
configurable lead-time before each close that matches its symbol. The calendar lives
here (regenerated by Strategy Lab from the NYSE / CME Globex calendars), so the EA never
needs a yearly code change.

## Raw URL (for the EA's WebRequest)

```
https://raw.githubusercontent.com/Chutithep39/mt5-market-schedule/main/early_closes.json
```

## Format

```jsonc
{
  "schema_version": 1,
  "generated_utc": "2026-06-20T06:24:16+00:00",
  "horizon_days": 400,
  "asset_classes": { "us_index": [...], "metals": [...], "energy": [...] },
  "early_closes": [
    {
      "date": "2026-07-03",
      "reason": "July 4th",
      "kind": "us_holiday",            // us_holiday | us_half_day | commodity_metals | commodity_energy
      "symbols": ["US500", "USTEC", ...],
      "close_et": "13:00",
      "close_broker": "2026-07-03 20:00", // ICMarkets server time = ET + 7h  -> MQL5 TimeCurrent()
      "close_utc": "2026-07-03T17:00:00+00:00" //                              -> MQL5 TimeGMT()
    }
  ],
  "maintenance": [
    {
      "reason": "ICMarkets server maintenance",
      "servers": "ALL MT5 Servers",
      "all_mt5": true,
      "start_broker": "2026-06-21 02:10", "end_broker": "2026-06-21 05:30",
      "start_utc": "2026-06-20T23:10:00+00:00", "end_utc": "2026-06-21T02:30:00+00:00"
    }
  ]
}
```

### Notes for the EA
- **Match by base symbol.** Your broker may add a suffix (e.g. `US500.cash`) — match by prefix.
- **`close_broker`** is ICMarkets server wall-clock (ET + 7h, constant). Compare against
  `TimeCurrent()`. **`close_utc`** compares against `TimeGMT()`.
- Regenerated automatically by Strategy Lab on startup; only pushed when the content changes.
